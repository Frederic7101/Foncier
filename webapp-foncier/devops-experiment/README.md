## Outil d'expérimentation Cursor + DevOps

Ce dossier contient un **cadre d'expérimentation** pour explorer l'intégration de Cursor avec des pratiques et outils DevOps, en prenant la webapp DVF existante comme projet cobaye.

- **Expression de besoins** et cahier des charges
- **Conception** (architecture, IHM, pipelines, sécurité, RGPD)
- **GED** (règles de rédaction, publication, versions, archivage)
- **Développement** (conventions branches/commits, lien au code)
- **Tests** (plan de tests, campagnes, visualisation des résultats)
- **Pilotage** (intégration JIRA / Confluence)
- **MCO** (ticketing support, rapports MCO)

### Arborescence

- `01-expression-besoins/` : modèles de cahier des charges, user stories, maquettage.
- `02-conception/` : templates d'architecture, IHM, pipelines, checklists sécurité/RGPD.
- `03-ged/` : conventions de nommage, gestion de versions, archivage.
- `04-developpement/` : conventions de branches, commits, points d'entrée du code cobaye.
- `05-tests/` : plan de tests, exemples, suivi et visualisation des campagnes.
- `06-pilotage/` : intégrations JIRA/Confluence (templates + scripts exemples).
- `07-mco/` : modèles de tickets et rapports MCO + exemples d'intégration.

Chaque sous-dossier est conçu pour être **réutilisable** sur d'autres projets que la webapp DVF.
