## Convention de nommage documentaire

### 1. Format général

Utiliser le format suivant pour les documents versionnés :

`DOC_TYPE_SUJET_vX.Y.ext`

Exemples :

- `SPEC_IHM-Stats_v1.2.md`
- `CONCEPTION_Architecture-DVF_v1.0.md`
- `PLAN_TESTS-Webapp_v0.9.md`

### 2. Types de documents

- `SPEC_` : spécifications fonctionnelles.  
- `CONCEPTION_` : conception technique.  
- `PLAN_` : plan (tests, déploiement, migration).  
- `RAPPORT_` : rapports (MCO, campagne de tests, audit).  

### 3. Numérotation de version

- `v1.0` : première version publiée / validée.  
- `v1.1` : correction mineure (orthographe, précision) sans changement de fond.  
- `v2.0` : évolution majeure (périmètre, règles de gestion, architecture).  

### 4. Compléments

- Ajouter éventuellement la **date** dans le contenu du document plutôt que dans le nom de fichier (évite les renommages fréquents).  
- Utiliser uniquement des caractères ASCII simples (`-`, `_`) dans les noms de fichiers pour éviter les problèmes de compatibilité.
