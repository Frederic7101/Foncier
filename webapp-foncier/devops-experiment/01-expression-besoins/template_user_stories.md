## Modèle de user stories / exigences

### Format recommandé

> En tant que **[acteur]**, je veux **[capacité]** afin de **[bénéfice]**.

Compléter chaque story avec :

- **ID** :  
- **Priorité** (Must / Should / Could / Won't) :  
- **Critères d’acceptation** (Given / When / Then) :  
- **Liens** (cahier des charges, ticket JIRA, maquette, plan de tests) :  

### Exemple (webapp DVF)

- En tant que **analyste immobilier**, je veux **visualiser l’évolution du prix/m² par commune sur une période donnée** afin de **comparer la dynamique de plusieurs territoires**.
